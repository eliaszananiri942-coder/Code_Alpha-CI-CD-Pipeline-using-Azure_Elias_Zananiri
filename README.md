# CodeAlpha CI/CD Pipeline using Azure

An automated, cloud-native Continuous Integration and Continuous Deployment (CI/CD) pipeline built to containerize and deploy a web application to **Azure App Service** via **Azure Container Registry (ACR)** using **GitHub Actions**.

## 🏗️ Architecture Overview

The pipeline implements a modern, secure, and highly efficient container deployment loop leveraging native cloud webhooks:

1. **Push**: Code changes are pushed to the `main` branch of the GitHub Repository.
2. **Build**: GitHub Actions triggers a workflow runner to build a isolated Docker image from the local `Dockerfile`.
3. **Publish**: The runner authenticates with a private Azure Container Registry (ACR) and pushes the compiled image with the `latest` tag.
4. **Deploy**: The runner executes a secure POST request to an Azure App Service continuous deployment Webhook.
5. **Pull**: Azure App Service uses stored registry credentials to pull the updated image layers and seamlessly switches traffic over to the live site.

---

## 🛠️ Tech Stack & Utilities

* **Host OS/Workspace:** Kali Linux (`device1`)
* **Version Control & CI/CD Platform:** GitHub & GitHub Actions
* **Cloud Provider:** Microsoft Azure
* **Container Engine:** Docker
* **Private Container Registry:** Azure Container Registry (ACR)
* **Application Hosting:** Azure App Service (Linux Web App Container)
* **API Ingestion Engine:** `curl` via automated deployment webhooks

---

## 🔑 GitHub Actions Configurations & Secrets

To run this pipeline securely without hardcoding credentials into version control, the following Repository Secrets must be configured in GitHub Settings:

| Secret Name | Description | Source / Extraction Command |
| :--- | :--- | :--- |
| `ACR_PASSWORD` | Administrative access token for pushing images to ACR. | `az acr credential show --name <registry-name>` |
| `AZURE_WEBAPP_WEBHOOK` | Private, unique deployment trigger URL generated by Azure App Service. | `az webapp deployment container config --enable-cd true` |

## 🚀 The CI/CD Workflow (`deploy.yml`)

The automation lifecycle is configured inside `.github/workflows/deploy.yml`:

```yaml
name: CodeAlpha CI/CD Pipeline

on:
  push:
    branches: [ "main" ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
    - name: Checkout Source Code
      uses: actions/checkout@v3

    - name: Log in to Azure Container Registry (ACR)
      uses: azure/docker-login@v1
      with:
        login-server: codealpharegistry99.azurecr.io
        username: codealpharegistry99
        password: ${{ secrets.ACR_PASSWORD }}

    - name: Build and Push Docker Image
      run: |
        docker build . -t codealpharegistry99.azurecr.io/codealpha-app:latest
        docker push codealpharegistry99.azurecr.io/codealpha-app:latest

    - name: Trigger Azure Container Deployment Webhook
      run: |
        curl -X POST "${{ secrets.AZURE_WEBAPP_WEBHOOK }}"
